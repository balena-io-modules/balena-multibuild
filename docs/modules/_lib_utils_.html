drainStream

  • drainStream(stream: Readable): Promise<void>
    • Defined in lib/utils.ts:20

    In the tar-stream implementation, there are bugs which can cause it to freeze if some of the entries are not drained, which can happen when splitting the tar stream into it's constiuent streams.

    This function will drain a stream and drop the output, returning a promise which resolves when the stream is drained.

    Parameters

    • stream: Readable

      A readable stream to be drained

    Returns Promise<void>

    A promise which resolves when the stream is drained
